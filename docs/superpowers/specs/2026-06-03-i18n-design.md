# 中英文国际化的支持 (i18n)

**日期:** 2026-06-03  
**项目:** Launchpad — API-first Solana launch workstation  
**范围:** 全部可见的 UI 以及 API 错误的消息 (客户端翻译)

---

## 目标

为当前中文网页添加英文切换能力，用户可通过顶栏按钮在中/英之间切换。

---

## 范围确认

| 范围 | 覆盖 | 来源文件 |
|---|---|---|
| 主工作台页面 | ✅ | `app/page.tsx` (641 行, `"use client"`) |
| Sign 页面 | ✅ | `app/sign/[id]/page.tsx` (server component, 读 cookie) |
| Wallet 检测消息 | ✅ | `lib/wallet/browser-wallet.ts` — 改为返回 code, 客户端映射 |
| Workbench-flow 辅助消息 | ✅ | `lib/launch/workbench-flow.ts` — 改为返回 code, 客户端映射 |
| API 错误消息 | ✅ | `lib/launch/validator.ts` etc. — 已有 `code`, 客户端 `t("errors."+code)` |

不翻译: 平台品牌名 (`pump.fun`, `Raydium LaunchLab`, `Meteora DBC`), 单位 (`SOL`, `lamports`), 专有名词 (`Provider`, `Base URL`, `Model`, `API Key`)。

## 实现方案: 轻量 DIY React Context + JSON

**不出新依赖。** 对比方案:
- next-intl: ~50kb, 需 middleware + i18n routing 配置, 对单客户端页过重
- URL 路由: 工作台不需要语言相关 URL

### 目录结构 (新增)

```
lib/i18n/
  provider.tsx        // I18nProvider + useI18n() hook
  messages.ts         // 翻译字典类型 + 全部 {zh/en} key-value
  detect.ts           // detectLocale() 函数
  LocaleToggle.tsx    // 顶栏语言切换按钮组件
```

### 数据流

```
[首次访问]
  navigator.language.startsWith("zh") → "zh", 否则 "en"
  localStorage.setItem("launchpad.locale", ...)

[后续访问]
  layout inline script → 读 localStorage → 设 <html lang> / data-locale
  I18nProvider mount → 读 localStorage → 选 messages[locale] → t()

[用户切换]
  LocaleToggle 点击 → setLocale(newLoc)
  → localStorage.setItem("launchpad.locale", newLoc)
  → document.cookie = "launchpad.locale=...; path=/; max-age=604800"
  → React re-render 全部 t() 更新

[Sign 页 (server component)]
  读 cookie "launchpad.locale" → 选 messages[locale] → 编译 inline
```

  // 从 messages[locale] 找 key, 找不到则返回 fallback 或 key 本身

### Layout 变化

`app/layout.tsx`:

```tsx
export default function RootLayout({ children }) {
  return (
    <html lang="zh-CN" suppressHydrationWarning>
      <head>
        <script dangerouslySetInnerHTML={{
          __html: `(function(){var l=localStorage.getItem("launchpad.locale");if(!l)l=navigator.language.startsWith("zh")?"zh":"en";document.documentElement.lang=l;})()`
        }} />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

`lang` 初始值在 layout 层面不影响实际渲染（client re-render 不覆盖 `<html>` 属性）, 所以从 `zh-CN` 改为随机的初始值也无妨。 inline script 会在 React hydration 前运行, 保证正确。

### Page 变化

`app/page.tsx`:
- Root JSX 用 `<I18nProvider>` 包住
- topbar `.pill-row` 末尾插入 `<LocaleToggle />`
- 全部中文 hardcoded 字符串改为 `t("key.path")`

`app/sign/[id]/page.tsx`:
- import messages, 读 cookie
- `const locale = cookies().get("launchpad.locale")?.value ?? "en"`
- `const t = (key: string) => messages[locale]?.[key] ?? key`
- 替换硬编码中文为 `t(...)`

## 边界 & 注意事项

- 探测 `navigator.language` 仅首次; 手动切换后存 localStorage 覆盖
- Cookie path="/" 保证 sign 页读到
- 命名 `launchpad.locale` 与现有 `launchpad.*` localStorage key 风格一致
- 只支持 `"zh"` 和 `"en"`; 不处理 `"zh-HK"`, `"zh-TW"` 等变体, 统一归入 `"zh"`
- 翻译 key 约定: 按组件/面板命名空间 + 点号, 如 `form.walletAddress`, `errors.wallet_invalid`
- `t()` 找不到 key 时回退到 key 本身的字符串 (开发阶段容易发现缺失项)
- 不改动任何 E2E 测试; 测试中使用 Vitest, 没有 DOM 依赖, 不需要 mock i18n
